# Study Plan

Project: `ai-tech-industry-investment-judgment`
Duration: 30 days
Progress format: `第X/30天`
Default daily time: flexible, with green/yellow/red protocol

## Compression Rule

本项目是“30 天压缩版”，不是“30 天删减版”。原 90 天内容全部保留：AI 技术判断、产业链推演、公司研究、财务估值、概率风控、模拟组合、产业监控和 Memo 输出都必须覆盖。

区别只在交付形态：

| Original 90-day style | 30-day compressed style |
|---|---|
| 慢速深挖多个案例 | 每天只抓一个高杠杆对象，快速形成可复用资产。 |
| 多篇完整 Memo | 3 篇压缩 Memo，其中 1 篇完整结构草案。 |
| 完整估值模型 | 轻量可改假设模型，字段必须完整但细节后续加深。 |
| 完整监控系统 | Dashboard v0.1，先有指标、来源、频率、告警规则。 |
| 模拟组合长期跟踪 | 建立规则和初始模拟组合，不假装有长期验证。 |

## Learning Strategy

这个项目采用“研究能力优先、投资动作暂缓”的策略。先建立可验证的技术、产业、财务、估值和概率判断力，再讨论资本配置。趋势正确不等于投资正确，产业增长不等于股东回报，模型分数不等于真实决策质量。

## Zero-Base Cognitive Chain

| Step | Project version |
|---|---|
| Observable problem | AI 时代信息、技术和资本开支变化太快，如何判断哪些机会真实、可持续、可投资、可复利。 |
| Minimum unit | 一个 claim、一个指标、一个产业节点、一家公司、一个现金流假设、一个风险约束。 |
| Representation | 证据等级、趋势图、产业链节点、财务模型、概率、情景、证伪条件。 |
| Mechanism | 技术进步改变能力和成本，资本开支流向瓶颈，瓶颈改变利润池，公司通过商业模式和资本结构把机会转成现金流。 |
| System flow | 技术事实 -> 产业需求 -> 供给弹性 -> 公司质量 -> 财务结果 -> 市场预期 -> 风险约束 -> 决策记录。 |
| Constraints/costs | 数据质量、预测不确定性、估值过高、融资风险、建设周期、政策、电力、竞争、执行风险。 |
| Failure modes | 把新闻当事实，把趋势当投资，把收入增长当 FCF，把 AI 热度当护城河，把高胜率叙事当低风险。 |
| Use cases | 职业定位、产品机会、研究输出、内容/咨询、模拟组合、长期投资框架。 |
| ROI action | 每天留下一个可复用资产：卡片、表格、模型、图谱、Memo、监控规则。 |
| Verification | 能解释、能计算、能写反方、能说明什么证据会改变判断。 |

## Green / Yellow / Red Protocol

| Mode | Time | Required action | Stop rule |
|---|---:|---|---|
| Green | 45-60 min | 主动回忆 + 一手资料 + 核心模型 + 实操 + Notion/GitHub 更新 | 留下一个可复用资产。 |
| Yellow | 15-25 min | 回忆一个概念 + 读一页或提取一条数据 + 写一张卡 | 不补全大任务，只保留连续性。 |
| Red | 5-10 min | 记录状态、卡点、明天第一步 | 停止，不补课，不制造伪进度。 |

## Scoring

| Score | Meaning |
|---:|---|
| 1 | 只认识术语。 |
| 2 | 能复述定义，但不能用于判断。 |
| 3 | 能解释机制和边界。 |
| 4 | 能用于真实研究问题、公司分析或风险判断。 |
| 5 | 能写成可辩护 Memo，能主动提出反方和证伪条件。 |
