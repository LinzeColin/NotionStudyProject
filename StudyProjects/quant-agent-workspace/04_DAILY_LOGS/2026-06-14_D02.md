## Core Explanation Notes

### What is it?

Evidence/Data Source Matrix 是 Quant Agent Workspace 的数据准入表。它不问“有没有数据”，而问：

- 来源是否可信；
- 时间戳是否完整；
- 当时是否真的可交易；
- 数据是否足够新；
- 发生冲突时系统应该降级、暂停还是阻断。

### Core theory

交易系统里的数据不是事实本身，而是带延迟、来源、采样方式、修订风险和可交易窗口的信息。一个能跑实盘前置流程的 Agent，必须区分四个时间：

1. `event_time`: 事件真实发生时间。
2. `published_at`: 来源公开发布时间。
3. `received_at`: 系统接收到的时间。
4. `tradable_at`: 策略最早能够合理交易的时间。

只要 `tradable_at` 不清楚，回测就可能有 lookahead bias；只要 freshness 过期，live signal 就应该停。

### How should I use it?

每个数据源先进入 source matrix，再进入策略。Agent 只能用通过 source gate 的数据生成信号；只能用通过 risk/data gate 的信号生成 `OrderIntent`。

### How it helps decisions, learning, and work

这个概念把“我看到了一个信息”变成“这个信息能不能支持真实决策”。它会强迫你区分研究线索、可验证证据、可交易信号和可审批订单意图。

### How it improves ability, productivity, thinking, or efficiency

它减少低质量数据带来的错误回测、假信号和无效研究，让 Agent 的工作从“收集很多东西”变成“只把可用证据推进到下一层”。
