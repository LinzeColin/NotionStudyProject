# Top 1% Practical Operator Strategy

## Core Thesis

For Notion, Windows, and Markdown, Top 1% does not mean knowing more features. It means having lower operational latency and better judgment about where information should go.

The capability stack is:

```text
Keyboard speed
-> capture speed
-> structure judgment
-> execution visibility
-> review loop
-> continuous simplification
```

## Ability Tree

| Layer | Capability | Why it matters | Evidence |
|---|---|---|---|
| OS layer | Window, clipboard, screenshot, file navigation | Removes mechanical friction | Timed drills |
| Input layer | Chinese/English, symbols, Markdown syntax | Keeps writing flow uninterrupted | Clean notes written quickly |
| Structure layer | Task / Note / Project / Reference classification | Prevents Notion from becoming a dump | Inbox processing quality |
| Database layer | Minimal useful properties and views | Enables retrieval and execution | Pages are easy to scan |
| Loop layer | Capture, clarify, execute, review, improve | Converts tool use into compounding system | Daily review evidence |
| Judgment layer | Know when not to automate or overbuild | Protects ROI | Defer list maintained |

## Non-Negotiable Habits

1. Every new item enters one trusted Inbox or one known destination.
2. Every task must have a next action.
3. Every project must have a current status and evidence.
4. Every knowledge page must answer: what is it, why it matters, how to use it, when it fails.
5. Every day ends with 5-minute review.

## Defer List

These are intentionally deferred until the manual loop works:

- Complex formulas.
- Multi-database relation systems.
- Notion API automation.
- Dashboard aesthetics.
- Large template imports.
- Widget-heavy pages.
- Full personal OS rebuild.

## Scoring Rubric

Daily score is 20 points:

| Category | Points |
|---|---:|
| Active recall before learning | 3 |
| Keyboard drill completed | 3 |
| Real input structured | 4 |
| Notion page/database improved | 4 |
| Review completed | 3 |
| Friction logged and fixed | 3 |

Interpretation:

| Score | Meaning |
|---:|---|
| 0-7 | Too theoretical or disrupted |
| 8-12 | Basic continuity |
| 13-16 | Practical progress |
| 17-20 | Top 1% sprint behavior |

## Operator Anti-Patterns

| Anti-pattern | Correction |
|---|---|
| Collecting templates | Build one page from actual use |
| Memorizing shortcut lists | Drill only high-frequency actions |
| Making many databases | Start with fewer categories and stronger classification |
| Pretty dashboard first | Evidence and retrieval first |
| Automating too early | Manual loop first, automation later |
