# Notion Layout Plan

## Page Title

Study Project: Notion Windows Markdown Operator

## Placement

Preferred target: `个人价值增长` database, `Status = Knowledge`.

Rationale: this project is a practical productivity and knowledge workflow project.

## Page Sections

1. Current Status
2. 7-Day Sprint Map
3. Today
4. Shortcut Map v1
5. Friction Log
6. Daily Review
7. GitHub Source of Truth

## Minimal Database Policy

Do not create new Notion databases on Day 0 unless necessary. Use one page first. Add databases later only if Day 2 evidence shows repeated structured data is needed.

## Suggested Day 2 Database Set

Only after D2:

| Database | Purpose | Minimum properties |
|---|---|---|
| Inbox | raw capture | Name, Type, Status, Source |
| Tasks | execution | Name, Status, Priority, Due, Project |
| Notes | knowledge | Name, Topic, Tags, Summary |
| Projects | outcomes | Name, Status, Goal, Next Action |

## Sync Rule

Daily sync from Notion to GitHub should store:

- summary,
- completed output,
- friction log,
- scores,
- next action,
- important links.

Full note body is optional and should be included only when useful.
