# Study Plan - 3+4 Day Sprint

Project: `notion-windows-markdown-operator`

Day 1 starts on 2026-06-14.

## Operating Principle

The first 3 days build the minimum complete operating stack. The next 4 days calibrate it against real behavior.

```text
D1 Windows/XiaoXin keyboard layer
D2 Markdown + Notion structure layer
D3 Loop engineering layer
D4 Calibration 1: speed and friction
D5 Calibration 2: real project workflow
D6 Calibration 3: knowledge workflow
D7 Final review and personal SOP
```

## Daily Plan

| Day | Date | Theme | Main output | Pass condition |
|---|---|---|---|---|
| D1 | 2026-06-14 | Windows/XiaoXin operating layer | Keyboard map + device config checklist | 80% high-frequency actions done without mouse |
| D2 | 2026-06-15 | Markdown and Notion structure | Inbox / Notes / Tasks / Projects structure | Convert 3 messy inputs into structured pages |
| D3 | 2026-06-16 | Loop engineering | Daily loop template and review template | Run capture -> clarify -> execute -> review once end-to-end |
| D4 | 2026-06-17 | Calibration: speed | Friction log and shortcut remediation | Reduce repeated friction by at least 30% |
| D5 | 2026-06-18 | Calibration: work/project | One real project managed through Notion | Every task has owner/status/next action/evidence |
| D6 | 2026-06-19 | Calibration: knowledge | One research/learning page rebuilt cleanly | Page has summary, taxonomy, examples, actions, review prompts |
| D7 | 2026-06-20 | Final SOP | Personal operator SOP + scorecard | User can run the loop without Codex scaffolding |

## Day 1 Detailed Plan

Time budget options:

| Mode | Minutes | Use when |
|---|---:|---|
| Minimum | 45 | Busy day, keep continuity |
| Standard | 90 | Default |
| Attack | 180 | High-energy day |

Day 1 sequence:

1. Active recall baseline: list current Windows/Notion/Markdown habits without looking up.
2. Configure Lenovo/XiaoXin: FnLock, power mode, touchpad gestures, display/eye comfort, input method.
3. Train core Windows shortcuts: window switching, screenshots, clipboard history, file navigation.
4. Build `Shortcut Map v1` in Notion or Markdown.
5. Run timed drill: capture screenshot, paste into Notion, annotate, move into the correct section.
6. Log friction and adjust Day 2.

## Dynamic Adjustment Rules

At the end of each day, update next day based on evidence:

| Signal | If true | Adjustment |
|---|---|---|
| Actual minutes < 45 | Continuity risk | Next day becomes recovery mode, keep one core output only |
| Energy <= 2 | Low learning quality | More drills, less theory |
| Recall score < 3/5 | Weak retention | Start next day with 15-minute active recall |
| Friction count > 5 | Workflow too heavy | Simplify Notion structure before adding features |
| Output missing | Not operational yet | Repeat output task before new concept |
| User finishes early | Capacity available | Add one real project or knowledge-base scenario |

## Stop Conditions

Do not add advanced Notion formulas, relations, automations, widgets, or API workflows until:

- D1 Windows layer is fast enough.
- D2 structure layer is understandable.
- D3 loop has been run once with real content.

## Final Deliverables

1. Shortcut Map v1.
2. Personal Notion operating page.
3. Inbox / Notes / Tasks / Projects structure.
4. Daily review template.
5. Weekly review template.
6. Friction log and fixes.
7. Final Top 1% Practical Operator SOP.
