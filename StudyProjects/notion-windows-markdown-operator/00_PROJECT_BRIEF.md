# Project Brief - Notion Windows Markdown Operator

Project slug: `notion-windows-markdown-operator`

Timezone: Australia/Sydney

Period: 2026-06-14 to 2026-06-20

Mode: 3+4 day Top 1% practical operator sprint

## Goal

Become a practical Top 1% operator for daily knowledge and execution workflow on Lenovo/XiaoXin Windows using Notion, Markdown, and keyboard-first workflows.

This project is not about memorizing every shortcut. The target is faster capture, clearer structure, less context loss, and reliable execution loops.

## Success Criteria

By 2026-06-20, the user should be able to:

| Capability | Practical standard |
|---|---|
| Capture | Put any task, note, screenshot, or idea into the right place in under 30 seconds |
| Structure | Convert messy input into Task / Note / Project / Reference in under 3 minutes |
| Operate Windows | Use keyboard-first switching, screenshots, clipboard history, file navigation, and window layout without breaking focus |
| Use Markdown | Write clean structured notes with headings, lists, tables, code blocks, links, and summaries |
| Use Notion | Build and maintain useful pages/databases without template bloat |
| Loop | Run daily capture, clarify, execute, review, and improve loop |
| Explain | Explain why the system works and when not to overbuild it |

## Top 1% Definition

Top 1% practical operator means high execution throughput with low coordination cost:

1. Low-friction capture.
2. Correct classification of information.
3. Fast page/database creation.
4. Keyboard-first operation.
5. Clear daily and weekly loops.
6. Evidence-based adjustment, not template collecting.
7. Ability to teach the workflow to another person.

## Constraints

- No heavy automation before the workflow is manually understood.
- No complex Notion formulas until the basic loop is stable.
- No template hoarding.
- No progress claims without evidence.
- GitHub stores durable state; Notion stores clean operating notes.
