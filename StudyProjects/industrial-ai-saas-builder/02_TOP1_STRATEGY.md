# Top 1% Strategy - Industrial AI SaaS Builder

## Realistic Top 1% Definition

The target is not absolute mastery of every field. The target is Top 1% cross-domain operating ability in this niche:

> Turning an industrial problem into an AI SaaS opportunity, architecture, execution plan, and validated decision memo.

## Ability Definition

| Ability | Top 1% behavior |
|---|---|
| Problem judgment | Separates real expensive pain from interesting but low-ROI automation. |
| AI judgment | Knows when RAG, agents, local LLMs, evals, or classic software should be used or avoided. |
| Business judgment | Connects workflow pain to buyer, budget, pricing, retention, and proof-of-value. |
| Architecture judgment | Can draw system boundaries, data flows, permissions, evals, observability, and failure recovery. |
| PM judgment | Converts vague goals into PRD, user stories, acceptance criteria, and cuttable releases. |
| Codex leverage | Uses Codex through run contracts, task packs, verification, and handoffs rather than vague prompts. |

## Defer List

| Defer | Reason |
|---|---|
| Heavy model training | Too slow and low ROI before product/data need is validated. |
| Full industrial digital twin | Too broad before a narrow workflow wedge exists. |
| Complex Notion dashboard | Notion should remain clean; GitHub holds structured state. |
| Multi-agent autonomy | Premature before evals, tool contracts, and boundaries exist. |
| Generic AI news chasing | Useful only if mapped to product or architecture decisions. |
