# AI Top 1% Strategy

Project: AI
Slug: `ai`

## Top 1% Capability Definition

Top 1% for this 30-day Study Project means practical, compounding judgment rather than academic completeness.

| Capability | Definition |
|---|---|
| Judgment | Know which AI capability fits which problem, and when not to use AI |
| Practical skill | Build prompts, workflows, tool-use patterns, and simple agent specs |
| Output quality | Produce reusable artifacts: maps, checklists, prompt packs, automation specs |
| Business/product sense | Connect AI to cost reduction, speed, quality, leverage, and new products |
| Frontier awareness | Track current model/tool/agent patterns without chasing noise |
| Safety/evals | Know failure modes, approvals, privacy, hallucination, and test design |

## High-ROI Expansion Map

| Board | Why It Matters | ROI Use Case | Example Output |
|---|---|---|---|
| AI fundamentals | Prevents shallow tool-chasing | Better model/tool decisions | AI concept map |

| Evals and reliability | Separates demos from useful systems | Avoid bad deployment decisions | Eval checklist |

| AI for research | Improves learning and decision speed | Reading, summarizing, synthesis | Research workflow template |
| AI-native personal OS | Compounds across all projects | Daily planning, review, memory | Personal AI operating loop |

## 20% High-Leverage Knowledge

1. Token, context, and model behavior.
2. Prompting as specification and interface design.
3. Retrieval, memory, and context engineering.
4. Tool use and function calling.
5. Agent loop: goal, planning, action, observation, state, evaluation.
6. Workflow automation: trigger, transform, action, approval, logging.
7. Evals: correctness, usefulness, robustness, regression checks.
8. Human-in-the-loop and approval gates.
9. Privacy, data boundaries, and prompt injection risks.
10. Product ROI: where AI reduces time, improves quality, or creates new leverage.

## 80% Defer List

| Defer | Reason |
|---|---|
| Training large models from scratch | Low ROI for current goal |
| Deep math of transformers first | Useful later, not first-order for practical leverage |
| Chasing every new model release | High noise unless tied to a use case |
| Building complex multi-agent systems too early | Often hides weak workflow design |
| Fully autonomous actions | Risky without evals, approvals, and logs |
| Heavy Notion dashboards | Can become system work instead of learning progress |

## Key Contrarian Points

- Most useful AI systems are workflow systems, not just prompts.
- Better context and evaluation often matter more than a newer model.
- Agents are not magic autonomy; they are controlled loops with state, tools, and failure handling.
- Automation without approval gates can create more risk than leverage.
- The best ROI comes from repeated workflows, decision bottlenecks, and output bottlenecks.

## Default Scope Recommendation

For 30 days, focus on:

```text
AI literacy -> LLM usage judgment -> tool use -> agent workflows -> automation portfolio
```

This is broad enough to compound across future projects, but narrow enough to produce practical outputs in one month.
