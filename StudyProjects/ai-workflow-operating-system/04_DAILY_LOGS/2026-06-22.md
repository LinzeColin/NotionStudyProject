## Session Goal

Start Prompt Engineering and Loop Engineering learning with the user's real Codex failure mode as the teaching object: agents running off-scope, forgetting baseline, hallucinating completion, and shifting control cost back to the user.

## Active Recall / Diagnostic Signal

The user identified three recurring problems from prior Codex development work:

1. Codex runs off-course as context and task complexity grow.
2. Codex forgets initial baseline and minimum acceptance standards under context pressure.
3. Codex partially understands intent, then produces uncontrolled or hallucinated output.

The user strongly rejected long manual prompt templates because they violate ROI, low-token, and user-friendly principles.

## Core Learning

Prompt Engineering is not about making the user write longer prompts. For the user's workflow, high-quality Prompt Engineering means reducing misunderstanding while keeping the user's input short.

Loop Engineering is not trusting an agent to remember rules. It is designing a low-cost loop where a fallible agent must repeatedly self-check goal, boundary, acceptance, stop conditions, and evidence.

## Corrected Mental Model

- Bad model: user writes longer and more complex prompts every time.
- Better model: durable rules, project state, and evidence surfaces carry the repeated control burden.
- Best practical model: Codex should automatically read existing `AGENTS.md`, `HANDOFF.md` or state, Task Pack, and acceptance files; then produce a compact execution contract before edits.

## Failure Observed During Teaching

Codex itself repeatedly demonstrated the target failure mode during this session:

- shifted from teaching into solution design;
- changed a local skill before the learning purpose was fully closed;
- prematurely claimed the lesson was complete;
- failed to proactively perform Study Project closeout until challenged by the user.

This is evidence that agent reliability cannot be based on trust or instruction presence alone.

## User Feedback

The user was dissatisfied with teaching that:

- transferred prompt complexity to the user;
- required the user to repeatedly correct the agent;
- claimed learning completion without checking Study Project acceptance requirements;
- failed to sync GitHub/Notion proactively.

## Corrected Principle

A useful AI workflow must be self-running and self-hosted as much as possible. The human should state the task goal in simple language. Codex should handle loop setup, prompt framing, output boundaries, acceptance standards, stop conditions, iteration mode, and evidence logging.
